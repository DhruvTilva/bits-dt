1. DFNN Forward & Backpropagation (Numerical) HIGH

Covered intensively: 08-03-2026 (full session numerical)
Why important: Prof spent the ENTIRE 08-03 session on DFNN numericals step-by-step. She said "first you need to understand the network and the weights" and worked through complete forward + backward pass with ReLU + Sigmoid. This is almost certainly an EC3 question.
  • Network Setup: From W1, W2, B1, B2 matrices → infer architecture: input neurons = cols(W1), hidden neurons = rows(W1) = len(B1), output neurons = len(B2)
  • Forward Pass Step 1: Z1 = W1 · X + B1 → apply ReLU → A1 = max(0, Z1)
  • Forward Pass Step 2: Z2 = W2 · A1 + B2 → apply Sigmoid → A2 = 1/(1+e⁻ᶻ²)
  • Loss Calculation: Binary Cross-Entropy = −[y·log(ŷ) + (1−y)·log(1−ŷ)]; Multi-class: −Σyᵢ·log(ŷᵢ)
  • Backward Pass: Compute δ at output layer → chain rule back through each layer → update W_new = W_old − η·∂L/∂W
  • ReLU Derivative: 1 if x > 0 else 0 (dead neurons when x ≤ 0)
  • Bias: One bias per neuron in feedforward; ONE bias per filter in CNN (key difference Prof emphasized)
Forward: Z¹ = W¹X + b¹, A¹ = ReLU(Z¹) Z² = W²A¹ + b², A² = sigmoid(Z²) Backward: δ² = A²−y (for sigmoid+cross-entropy) δ¹ = (W²)ᵀδ² ⊙ ReLU'(Z¹) Weight update: W ← W − η·(δ·Aᵀ_prev)
Prof's Exam Tip: "When given a question, first identify the network architecture from the weight matrix dimensions. W1 is (hidden × input), W2 is (output × hidden), B1 has hidden-count elements, B2 has output-count elements."

2. CNN: Cross-Correlation, Output Size, Padding & Stride HIGH

07-03-2026 & 16-05-2026 (Prof said exam will have kernel questions)
Why important: Prof repeatedly worked through cross-correlation numericals across multiple sessions. In the 16-05 lecture she directly said: "In your exam, maybe you will be given some input dataset, it will have your kernels and you are going to compute the kernels as well as output. Check out how they are behaving." Direct exam signal.
  • Cross-Correlation Formula: (I ★ K)(i,j) = Σₘ Σₙ I(i+m, j+n) · K(m,n)
  • Output Size Formula: out = ⌊(N − F + 2P)/S⌋ + 1 where N=input, F=filter, P=padding, S=stride
  • Bias in CNN: ONE bias per filter (not per neuron!) — Prof explicitly emphasized this
  • Bias purpose: Prevents feature map values from going to zero, especially when filter values cancel out
  • Max Pooling: Subsamples/reduces spatial dimensions; picks max value in each window
  • Average Pooling: Takes mean; used in GoogLeNet/NIN global average pooling (GAP) instead of FC layers
  • Image invariance: Object of interest can appear anywhere → kernel slides across entire image
  • Parameter efficiency: Convolution shares weights → far fewer parameters than fully-connected
Output size: out = floor((N − F + 2P) / S) + 1 Example: Input 6×6, Filter 3×3, Stride=1, Padding=0 out = (6 − 3 + 0)/1 + 1 = 4×4 With padding=1: out = (6 − 3 + 2)/1 + 1 = 6×6 (same padding)
Prof's Exam Hint (16-05): "In your exam, you will be given a dataset with kernels, and you are going to compute kernels as well as output. Different scenarios — which kernel gives what output."

3. CNN Architectures: AlexNet, VGG, GoogLeNet (Inception), ResNet HIGH

21-03-2026 · Multiple lectures
Why important: These are the "deep CNN architectures" from Session 7 — a core syllabus module. Prof did detailed walkthrough of NIN, Inception, ResNet blocks with design philosophy. EC3 covers all topics.
  • LeNet-5: Oldest; input 32×32 → Conv → Sub-sample → Conv → Sub-sample → FC → FC → 10 classes. Historic significance.
  • AlexNet: First deep CNN winner (ImageNet 2012). Used ReLU (not tanh), Dropout for regularization, data augmentation, GPU training. 5 conv layers + 3 FC layers.
  • VGGNet: "Networks using blocks" — key design: use small 3×3 filters only, increase depth. VGG-16 (16 layers). Simple and uniform design.
  • NIN (Network in Network): Replaces linear conv filters with micro-MLPs using 1×1 convolutions. Global Average Pooling (GAP) replaces FC layers → reduces overfitting. Inspired Inception.
  • GoogLeNet (Inception): Inception blocks use multiple parallel filter sizes (1×1, 3×3, 5×5) + max pooling in parallel → concatenate. 1×1 bottleneck convolutions reduce computation. 22 layers. NO fully connected layer at end (uses GAP).
  • ResNet: Residual blocks: F(x) + x (skip connection). Solves vanishing gradient in very deep networks. Can train 100+ layer networks. Identity shortcut means gradient flows directly.
  • Key Design Principle — ResNet: H(x) = F(x) + x → learning residual is easier than learning full mapping
  • Transfer Learning: Freeze early layers (general features: edges, textures), fine-tune later layers. Much less data needed.
ResNet Residual Block: Output = F(x, {Wᵢ}) + x where F = 2 conv layers with BN + ReLU x = identity shortcut connection Inception Block (parallel branches): [1×1 conv] + [1×1→3×3 conv] + [1×1→5×5 conv] + [MaxPool→1×1] → Concatenate all outputs along channel dim

4. GRU (Gated Recurrent Unit) — Theory + Numerical HIGH

11-04-2026 (Full GRU numerical session)
Why important: Prof spent a full session on step-by-step GRU numerical walkthrough with sentence "I am tired". She derived Z_t, R_t, H̃_t, H_t manually. Very likely to appear numerically in EC3.
  • Motivation: Vanilla RNN suffers vanishing gradient & can't handle long-range dependencies. GRU adds gating mechanism.
  • GRU has 2 gates: Update Gate (Z) and Reset Gate (R) — simpler than LSTM's 4 gates
  • Update Gate Z_t: Controls how much of OLD hidden state to keep vs new candidate state (interpolation between past & new info)
  • Reset Gate R_t: Controls how much of PAST hidden state influences new candidate. σ=0 → ignore past; σ=1 → use all of past.
  • Candidate Hidden State H̃_t: tanh(WH·Xt + UH·(Rt ⊙ H_{t-1}))
  • Final Hidden State H_t: Interpolation: H_t = (1 − Z_t) ⊙ H_{t-1} + Z_t ⊙ H̃_t
  • Key insight (from numerical): When Z_t ≈ 0 → keep old info; when Z_t ≈ 1 → accept new candidate; values ~0.52 mean balanced update
  • Parameter sharing: Same weights used at every time step (unlike feedforward)
GRU Equations: Z_t = σ(Wz·Xt + Uz·H_{t-1}) ← Update gate R_t = σ(Wr·Xt + Ur·H_{t-1}) ← Reset gate H̃_t = tanh(WH·Xt + UH·(R_t ⊙ H_{t-1})) ← Candidate H_t = (1 − Z_t) ⊙ H_{t-1} + Z_t ⊙ H̃_t ← Final state Numerical (from lecture, sentence "I am tired"): Word embeddings: I=0.2, am=0.3, tired=0.8 H₀ = 0, weights: Wz=0.5, Wr=0.4, WH=0.6, Uz=0.1, Ur=0.2, UH=0.3 For t=1 (word "I", X=0.2): Z₁ = σ(0.5×0.2 + 0.1×0) = σ(0.1) ≈ 0.525 R₁ = σ(0.4×0.2 + 0.2×0) = σ(0.08) ≈ 0.52 H̃₁ = tanh(0.6×0.2 + 0.3×0.52×0) = tanh(0.12) ≈ 0.12 H₁ = (1-0.525)×0 + 0.525×0.12 ≈ 0.063

5. LSTM (Long Short-Term Memory) — Structure & Gates HIGH

11-04-2026 · Referenced across sessions
Why important: LSTM is one of the core RNN architectures in Session 10. Prof mentioned "focus was GRU and LSTM". Both together are almost certain to be tested.
  • LSTM has 4 gates: Forget gate (f), Input gate (i), Cell gate (g), Output gate (o)
  • TWO states: Cell state (C_t) = long-term memory; Hidden state (H_t) = short-term memory/output
  • Forget gate f_t: σ(Wf·[H_{t-1}, X_t] + bf) → decides what to forget from cell state
  • Input gate i_t: σ(Wi·[H_{t-1}, X_t] + bi) → decides what new info to store
  • Cell gate g_t: tanh(Wg·[H_{t-1}, X_t] + bg) → candidate new values
  • Cell state update: C_t = f_t ⊙ C_{t-1} + i_t ⊙ g_t
  • Output gate o_t: σ(Wo·[H_{t-1}, X_t] + bo) → filters cell state for output
  • Final output: H_t = o_t ⊙ tanh(C_t)
  • GRU vs LSTM: GRU fewer params (2 gates vs 4), trains faster; LSTM better for very long sequences; GRU often matches LSTM in practice
LSTM Gates: f_t = σ(Wf·[H_{t-1}, Xt] + bf) ← Forget gate i_t = σ(Wi·[H_{t-1}, Xt] + bi) ← Input gate g_t = tanh(Wg·[H_{t-1}, Xt] + bg) ← Cell gate o_t = σ(Wo·[H_{t-1}, Xt] + bo) ← Output gate C_t = f_t ⊙ C_{t-1} + i_t ⊙ g_t ← Cell state H_t = o_t ⊙ tanh(C_t) ← Output

6. Attention Mechanism + Transformer Architecture HIGH

25-04-2026 · 08-05-2026 · 09-05-2026
Why important: Prof spent 3+ sessions on attention → transformers. She covered Q/K/V, self-attention, multi-head attention, masked attention, cross-attention, positional encoding, encoder-only/decoder-only/encoder-decoder variants. Most modern/exam-relevant topic.
  • Why Transformers: RNNs can't parallelize (sequential), have vanishing gradient, fixed-size hidden state bottleneck. Transformers solve all three.
  • Attention = Information Retrieval: Given a Query, find relevant Keys, return weighted Values
  • Scaled Dot-Product Attention: Attention(Q,K,V) = softmax(QKᵀ/√d_k)·V
  • Why divide by √d_k: Prevents dot products from getting very large → softmax becomes saturated (very sharp) → gradients vanish
  • Self-Attention: Q, K, V all come from same input sequence. Every word attends to every other word simultaneously.
  • Multi-Head Attention: Run h parallel attention heads → each learns different aspects of relationships → concatenate → linear transform
  • Cross-Attention: Q from decoder, K and V from encoder output. Used for machine translation.
  • Masked Attention: Future tokens masked (set to −∞ before softmax). Used in decoder during training. GPT uses this.
  • Positional Encoding: Transformers have no recurrence → no position info. Add sin/cos positional encoding to embeddings.
  • Encoder-only (BERT): Bidirectional; classification, NER, sentiment
  • Decoder-only (GPT): Autoregressive text generation; masked self-attention
  • Encoder-Decoder (T5): Machine translation, summarization, Q&A
  • Transformer Block components: Multi-head self-attention + Layer Norm + Feed-Forward + Layer Norm (residual connections throughout)
  • Pre-training + Fine-tuning: Pre-train on massive unlabeled data (general language understanding) → fine-tune on specific task with labeled data
  • LoRA (Low-Rank Adaptation): Parameter-efficient fine-tuning — update < 1% of parameters instead of full fine-tuning
Scaled Dot-Product Attention: Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V Multi-Head Attention (h heads): head_i = Attention(QWiQ, KWiK, VWiV) MultiHead(Q,K,V) = Concat(head₁,...,headₕ) · W_O Transformer Encoder Block: 1. x' = LayerNorm(x + MultiHead(x,x,x)) ← Self-Attention + residual 2. out = LayerNorm(x' + FFN(x')) ← FeedForward + residual Attention head pruning: A transformer with 16 heads, 5 redundant heads Memory saved = 5 × 12 MB = 60 MB Remaining heads = 16 − 5 = 11
Prof's Exam Tip (16-05): "If redundant heads need to be removed, and the question asks memory used by redundant heads — go by the question wording. 5 heads × 12MB = 60MB. If asked to optimize further, justify by saying we can keep 1 of the 5 and remove 4, leaving 12 heads. Justify your answer always."

7. Optimization Algorithms — SGD, Momentum, Adam (Numerical) HIGH

09-05-2026 (Full session numerical — SGD → Momentum motivation)
Why important: Prof did a detailed multi-iteration numerical on mini-batch gradient descent with L(W) = W₁² + 4W₂², showing how W₂ converges much faster (steeper gradient). This motivates Momentum and adaptive methods. Numerical very likely in EC3.
  • Gradient Descent: W ← W − η·∇L. Problem: one learning rate for all params → unbalanced convergence
  • SGD Problem: Large learning rate → oscillation (zigzag); small learning rate → very slow convergence
  • Momentum: Accumulate past gradients → smoother update direction. Reduces oscillation in steep directions.
  • Momentum formula: v_t = β·v_{t-1} + (1−β)·g_t; W ← W − η·v_t. β typically 0.9
  • Adagrad: Per-parameter adaptive learning rate. Divides η by √(sum of squared gradients). Issue: learning rate decays to zero
  • RMSProp: Fixes Adagrad — uses exponential moving average of squared gradients instead of sum. v_t = β·v_{t-1} + (1−β)·g_t²; W ← W − η/√(v_t+ε)·g_t
  • Adam: Combines Momentum (1st moment) + RMSProp (2nd moment) with bias correction. Most popular optimizer.
  • From lecture observation: W₂ (steeper gradient coefficient 8) converges MUCH faster than W₁ (coefficient 2). This motivates adaptive methods.
SGD Update: W_new = W_old − η · ∇L(W_old) Numerical (L = W₁² + 4W₂², start: W₁=4, W₂=2, η=0.1): Iteration 0: Loss = 4² + 4·2² = 16+16 = 32 ∂L/∂W₁ = 2W₁ = 8, ∂L/∂W₂ = 8W₂ = 16 W₁_new = 4 − 0.1×8 = 3.2 W₂_new = 2 − 0.1×16 = 0.4 Iteration 1: Loss = 3.2² + 4×0.4² = 10.24+0.64 = 10.88 → W₂ converges much faster (steeper gradient)! Adam: m_t = β₁·m_{t-1} + (1−β₁)·g_t ← 1st moment (momentum) v_t = β₂·v_{t-1} + (1−β₂)·g_t² ← 2nd moment (RMSProp) m̂_t = m_t/(1−β₁ᵗ), v̂_t = v_t/(1−β₂ᵗ) ← Bias correction W ← W − η·m̂_t / (√v̂_t + ε) ← Update

8. Regularization — Dropout, Batch Norm, L1/L2, Overfitting HIGH

12-05-2026 (Last topic lecture) · 16-05-2026 (exam numericals)
Why important: Prof started the 12-05 session: "So our last topic, last but not least, regularization." She structured it comprehensively. The 16-05 session had direct EC3-style questions: "CNN training 94% vs test 68% — identify and mitigate overfitting." This is a guaranteed exam topic.
  • Regularization goal: Prevent model from memorizing training data → improve generalization to unseen data
  • When needed: When model OVERFITS (high training accuracy, low test accuracy)
  • Underfitting = High Bias: Model too simple, can't capture the pattern. Fix: more layers, more neurons
  • Overfitting = High Variance: Model memorizes training data, fails on test. Fix: regularization
  • L2 Regularization (Ridge/Weight Decay): Add λ/2·Σw² to loss. Penalizes large weights. Shrinks weights toward zero (but not exactly zero).
  • L1 Regularization (Lasso): Add λ·Σ|w| to loss. Produces sparse weights (some exactly zero). Feature selection effect.
  • Dropout: Randomly zero out p fraction of neurons during training. Prevents co-adaptation. At test: scale by (1-p). Effective ensemble of 2^n networks.
  • Batch Normalization: Normalize activations of each layer (mean=0, var=1), then scale with learned γ,β. Reduces internal covariate shift, allows higher learning rates, slight regularization.
  • Layer Normalization: Like BN but normalizes across features (not batch). Used in Transformers. Works for variable-length sequences.
  • Regularization trade-off: May REDUCE training performance but IMPROVES test performance (intentional bias)
L2 Regularized Loss: J_reg = J + (λ/2)·Σ w² L1 Regularized Loss: J_reg = J + λ·Σ |w| Dropout (training): randomly set neuron output to 0 with prob p Dropout (test): multiply activations by (1 − p) Batch Norm: x̂ = (x − μ_B) / √(σ²_B + ε) y = γ·x̂ + β (learned scale and shift) Overfitting diagnosis (from 16-05 lecture): Training accuracy: 94%, Validation: 92%, Test: 68% → Large gap between validation & test = distribution shift + severe overfitting → Dropout alone won't solve distribution shift → Fix: data augmentation, more regularization, ensure test/train same distribution
Prof's Exact Exam Tip (16-05): "Would adding dropout ALONE solve this issue? NO — give 2 or more answers. Even if only 2 marks given, don't stop at 2 marks. If your answer is very good, you will get more."

9. RNN Architecture & Backpropagation Through Time (BPTT) MEDIUM

04-04-2026
Why important: Session 9 topic. Forms the foundation for GRU/LSTM questions. Prof walked through sequence modeling motivation, why FFN fails for sequences, and RNN's memory concept.
  • RNN Key Advantage: Variable-length input sequences, parameter sharing across time steps, memory via hidden state
  • Hidden State: H_t = tanh(Wx·X_t + Wh·H_{t-1} + b)
  • Output: Y_t = softmax(Wy·H_t + by)
  • RNN Problems: (1) Cannot parallelize — sequential computation; (2) Vanishing gradient for long sequences; (3) Fixed-size hidden state bottleneck
  • Vanishing Gradient: Gradient multiplied by Wh at each step → if |Wh|<1, gradient vanishes over many steps; if |Wh|>1, explodes
  • BPTT: Backprop through time — unroll RNN and apply chain rule across all time steps. Computationally expensive for long sequences.
  • Short-term vs Long-term dependency: "The clouds are in the sky" — short-term. "I grew up in France, so I speak fluent ___" — long-term (7 words back)
  • Bidirectional RNN: Process sequence forward AND backward → capture context from both directions
RNN Hidden State: H_t = tanh(Wx·Xt + Wh·H_{t-1} + b) RNN Output: Y_t = softmax(Wy·H_t + by) BPTT Gradient (simplified): ∂L/∂Wh = Σ_t ∂L_t/∂H_t · ∏(Wh·diag(tanh'(H_k))) → Products of Wh across time → vanishing/exploding gradient

10. Encoder-Decoder Architecture & Context Bottleneck MEDIUM

18-04-2026 · 08-05-2026
Why important: Session 9-10 topic. Prof detailed the context bottleneck problem and word embeddings / dimensionality with examples. Transition to attention was motivated by encoder-decoder limitations.
  • Encoder: Takes variable-length input → produces FIXED-size context vector C (e.g., 512 dims)
  • Decoder: Uses context C to generate variable-length output. Y2 depends on Y1, etc.
  • Context Bottleneck Problem: All input information compressed into ONE fixed vector → information loss for long sequences
  • Word Embeddings: Each word represented as multi-dimensional vector (512 dims in practice). More dimensions → more expressive representation.
  • Teacher Forcing: During training, feed correct previous output to decoder (not its own prediction). Speeds up training but creates train/test mismatch.
  • Attention solves bottleneck: Decoder can attend to ALL encoder hidden states (not just final one) → no information bottleneck

⚡ Common Mistakes to Avoid

  • Confusing L1 (sparse) with L2 (smooth) regularization
  • Applying Dropout during inference (don't!)
  • Mixing up Batch Norm (batch-wise) vs Layer Norm (feature-wise)
  • CNN bias: 1 per filter, NOT 1 per output neuron
  • GRU has 2 gates, LSTM has 4 — don't swap
  • Masked attention: prevents looking at FUTURE tokens
  • Cross attention: Q from one, K/V from another source
  • Adam gradient: divide by √v̂ + ε (bias-corrected!)
